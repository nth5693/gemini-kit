# Primary Workflow

## The `/cook` Workflow

```
/cook [task description]
```

### Steps

1. **Planning** - `/plan`
2. **Scouting** - `/scout`
3. **Coding** - Implement
4. **Testing** - `/test`
5. **Reviewing** - `/review`
6. **Committing** - `/git`

## Quick Workflows

| Task | Command |
|------|---------|
| New feature | `/cook implement` |
| Bug fix | `/cook fix` |
| Refactoring | `/cook refactor` |
| Full audit | `/cook full review` |

## When to Use

- **Simple change**: `/code` or `/fix`
- **Complex feature**: `/cook`
- **Code quality**: `/review`
- **Explore**: `/scout`
